# Fix version variables to align with new system davit.config

## Description

The `_dl_detect_context()` function in `davit-logger.sh` reads the project version
from `.env` by grepping for `^VERSION=`. The new standardised `.env` template
(generated by `generate-env` v0.1.2+) uses `PROJECT_VERSION=` instead of `VERSION=`.

This mismatch causes `D_PRJ_VER` to remain `"unknown"` in all log output for any
project using the new `.env` format — even when the version is correctly set.

**Observed:** `install.sh` dry-run for `manage-project-requirements` showed:
```
| manage-project-requirements | unknown | MODE:dev |
```
Expected: `| manage-project-requirements | 0.1.0 | MODE:dev |`

**Root cause:** `davit-logger.sh` line ~120:
```bash
D_PRJ_VER=$(grep '^VERSION=' "${src}" | cut -d'=' -f2)
```
Must also check `PROJECT_VERSION=` to cover the new `.env` format.
